# Threat model

VeilForge focuses on deterministic disclosure and privacy-migration risks, including public getters, event disclosure, plaintext calldata, weak authorization boundaries, and sensitive cross-contract flows.

It does not claim complete coverage of reentrancy, economic attacks, compiler bugs, governance risk, oracle manipulation, bridge risk, or production operational security.

A clean report means only that the current rule set did not match. It does not prove that a contract is secure or private.
